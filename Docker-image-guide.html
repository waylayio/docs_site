    Demo docker image

    A demo version of the Waylay platform is available as docker image. The image contains the following components:

    • Web interface
    • Inference engine
    • Single nodejs instance for sensor/actuator execution
    • REST interface

    What is missing compared to our paas offering:

    • No official support (only via forum), no bug fixes
    • There is only a single, hard coded account
    • No Broker support
    • No Cloud Cache support

    How to get started

    Follow the steps at http://docs.waylay.io/ to start a container on your local environment. We expect you to have boot2docker installed.

    In case you previously started the container and would like to pull in the latest version you can use this command:

    docker pull waylay/demo

    Web interface

    The web interface can be opened with the following command

    OS X

    open http://$(boot2docker ip):8080/

    Linux

    open http://localhost:8080/

    Windows

    boot2docker ip
start http://[ip returned by above command]:8080/

    log in with user and password demo/demo

    Import sensors and actuators from github

    Once you logged in, go to the public repo menu and download sensors and actuators into your application.

    Connecting to the REST api

    The rest api is available at http://[host]:8080/api. All requests need http basic authentication with username and password demoKey/demoSecret

    example

    curl --user demoKey:demoSecret http://$(boot2docker ip):8080/api/sensors

    Pushing data

    Since the broker is not available in the docker image we provide a minimal implementation with a single http call.

    POST http://[host]:8080/api/resources/{resourcename}

    example

    curl --user demoKey:demoSecret \
  -H "Content-Type: application/json" \
  -X POST \
  -d '{"temperature":23.35, "humidity":45}' \
  http://$(boot2docker ip):8080/api/resources/livingroom

    For more info on this api call check the broker/storage documentation.
